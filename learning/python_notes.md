# Python Notes

Python/environment-specific lessons and decisions for this project. See [Project Learning.md](Project%20Learning.md) for the full index.

## Lessons Learned

- **2026-07-12 — Python's `python` command was not resolving on Windows, so the environment had to be created with `py -m venv` and the venv interpreter used directly.** Fix: create the environment with `py -m venv .venv`, then use `.venv\Scripts\python.exe` for installs and execution (don't rely on `python`/`pip` resolving correctly on PATH in this environment).
- **2026-07-14 — `requirements.txt` was empty despite the `.venv` already having playwright, python-dotenv, reportlab, openai, pandas, and more installed** (from earlier untracked prototype work — see [architecture_notes.md](architecture_notes.md)). A fresh clone had no way to reproduce the environment. Fixed by running `.venv\Scripts\python.exe -m pip freeze > requirements.txt`. Lesson: when adopting a project with an existing `.venv`, check `pip freeze` against `requirements.txt` early — don't assume they're in sync just because both exist.
- **Import convention for this project: `from src.X.Y import Z`, run from the project root.** There's no `pyproject.toml`/`setup.py` — the project root being on `sys.path` (which happens automatically when running a script from the project root, e.g. `python src/rental_agent.py` or via pytest's rootdir detection) is what makes `src` resolve as a namespace package. Don't use bare `from X.Y import Z` (without the `src.` prefix) inside `src/` modules — it won't resolve the same way.
- **2026-07-14 — `tests/__init__.py`, deleted on 2026-07-13 as unexplained mystery-batch cruft (see [architecture_notes.md](architecture_notes.md)), turned out to actually be needed.** `python -m unittest discover -s tests -t .` fails with `ImportError: Start directory is not importable` without it — `unittest discover` requires the start directory to be an importable package when the top-level dir differs from it, unlike `src/` (a namespace package works fine there because nothing does an equivalent importability check on it). Restored it, and added `tests/storage/__init__.py` for the same reason when adding that subpackage. Test command: `.venv\Scripts\python.exe -m unittest discover -s tests -t .` from the project root.
