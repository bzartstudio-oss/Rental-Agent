## 2026-07-14 — V1.0 architecture: Rental Intelligence Platform

Reframed the project from a simple scraper to a self-improving research platform around 7 core principles (never lose information, permanent-database updates, versioned history, reproducible searches, extensible filters, extensible platforms/cities without redesign, business logic independent of any website). Rewrote `docs/00`–`10` with a concrete V1.0 design: SQLite as the storage engine, a Connector/Collector split, an immutable `search_results` snapshot table, a resolved legacy-`src/`-folder reconciliation plan, and a phased roadmap (storage first, second connector last, to validate the platform-independence boundary). Full decision log in [../learning/architecture_notes.md](../learning/architecture_notes.md) and [../learning/database_notes.md](../learning/database_notes.md). Also found and fixed more casualties from the 2026-07-13 file-wipe incident (`src/connectors/README.md` and several other empty `README.md` files that had gone unnoticed).

## 2026-07-14 — Phase 1 & 2: storage foundation and platform discovery

Implemented and tested the first two roadmap phases. Phase 1: SQLite schema, `storage/models.py`, and three repositories, with a test proving price history is appended rather than overwritten on re-observation. Phase 2: `discovery/platform_registry.py` and `discovery/discovery_agent.py`, tested against a registered (not yet real) seed platform. Along the way, executed the legacy-folder reconciliation from `docs/02_Folder_Guide.md` — deleting the stale prototype (`config/settings.json`, `src/config/`, `src/model/`, `src/models/`, `main.py`) after confirming with the user it didn't reflect the current design, rather than migrating it as originally planned. Fixed an empty `requirements.txt` and rediscovered that `tests/__init__.py` (deleted in the 2026-07-13 incident cleanup) was actually required for `unittest discover` to work. 12 tests passing. Both commits pushed to the `platform-v1` branch.

## 2026-07-14 — Phases 3–7: the rest of the pipeline, end-to-end

Built the remaining five roadmap phases in one session, completing the V1.0 architecture. No real rental platform had been picked yet, so rather than block on that product decision (or unilaterally scrape a real commercial site without checking its ToS), built and proved everything against two real reference connectors (`demo_platform.py`, `demo_platform_two.py`) — real Playwright fetches and real BeautifulSoup parsing of local HTML fixtures standing in for a live site. Phase 3: collectors (raw page store, image collector, HTTP collector), proven with a real fetch of `https://example.com`. Phase 4: the connector contract, the first reference connector, the extensible search-criteria registry, the Analysis Engine (plus an added `analyzers/engine.py` to compose the pipeline's write sequence), and `core/agent.py` — the real orchestrator, proven with a real end-to-end run producing real database rows. Phase 5: ranking, an HTML report generator (plain string templating, not Jinja2 — not an installed dependency), and `ui/cli.py`, the real entry point; `rental_agent.py` retired its old OpenAI-key-check stub in favor of delegating to the CLI. Phase 6: proved re-running the same search after a real fixture change accumulates history instead of overwriting it. Phase 7: a second connector with a deliberately different HTML shape, proving the platform-independence boundary holds without touching analyzers/ranking/storage/services.

Along the way: found and fixed a test-isolation bug (tests were silently writing into real project `data/raw_pages/`), cleaned up more pre-architecture leftovers (`output/exports/pdf/html/`), and ran the full CLI for real once against actual project data to get a genuine, non-test proof-of-concept report. 56 tests passing. What's left is not architectural — pick a real first platform and write one more connector for it.

## 2026-07-14 — Version 1.1: Multi-Platform Discovery Framework

Explicit user instruction: build platform discovery/management, not a real connector yet. Reworked the `platforms` schema (dropped `base_url`/`connector_module`/`is_active`; added `country`, `supported_cities`, `rental_types`, `homepage`, `search_url`, `requires_login`, `connector_available`, `connector_name`, `last_verified`, `discovery_method`). Split `DiscoveryAgent` into a search-facing `discover()` and a management-facing `sync_platforms()` implementing all 5 required behaviors (load existing, detect duplicates, update metadata, save new, mark unsupported). Seeded `discovery/known_platforms.py` with the 2 reference connectors plus 6 real, well-known rental platforms across 4 countries (Zillow, Apartments.com, Rightmove, Idealista, Fotocasa, ImmoScout24) as catalogued-but-unsupported — real public names/homepages, zero live requests made to compile the list. `ui/cli.py` now syncs the known-platforms list on every startup instead of a one-off registration. Updated all existing tests for the new `Platform` shape and added 17 new ones (73 total) covering the five discovery behaviors plus data-integrity checks on the seed list. Deleted the dev database (schema change, no migrations framework yet) and re-verified with a real CLI run: 8 platforms registered, 2 usable, 6 catalogued for future connectors. Full detail in [../learning/architecture_notes.md](../learning/architecture_notes.md).
